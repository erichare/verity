# Changelog

All notable changes to the `verity-x3p` codec (the Rust crate and its Python/R
bindings) are documented in this file. Releases are tagged `v<version>`;
pushing a tag runs `.github/workflows/release.yml`, which publishes the crate
to crates.io and the Python wheels + sdist to PyPI.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.2.0] - 2026-07-01

First packaged release: the `verity-x3p` crate on crates.io and the
`verity-x3p` wheel (module `verity_x3p`) on PyPI.

### Added

- Native Rust reader/writer for the X3P surface topography format
  (ISO 25178-72 / ISO 5436-2): the zip/OPC container with `main.xml` metadata
  and the `bindata` height matrix.
- All four ISO Z-data encodings on read (`I`/`L`/`F`/`D` — int16, int32,
  float32, float64) in the spec's column-major, X-fastest matrix order; float
  encodings (`F`/`D`) on write so invalid points round-trip as NaN.
- MD5 checksum validation on read (on by default, opt-out via `ReadOptions`
  for known-corrupt files) and checksum generation on write (point-data
  checksum in `main.xml` plus `md5checksum.hex`).
- Invalid-point handling: NaN heights plus an explicit validity mask.
- Path and in-memory APIs: `read_x3p` / `read_x3p_bytes` and `write_x3p` /
  `write_x3p_to_bytes`.
- Python binding via PyO3/maturin (abi3, CPython 3.9+): NumPy arrays of shape
  `(ny, nx)`, bit-identical round-trips with the Rust core.
- R binding (`verityx3p`, via rextendr) with a vendored copy of the core crate
  regenerated by `bindings/r/vendor-core.sh`, so the package is
  self-contained for CRAN-style builds.
